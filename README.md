# Year-to-Year Bike Share Analysis | SQL Project

## Project Overview

This project focuses on performing an in-depth Year-to-Year Bike Share Analysis using SQL to uncover rider behavior, business performance, seasonal trends, profitability, and operational insights from bike-sharing data.

The project combines multiple datasets, applies advanced SQL techniques, and performs business-driven analysis to generate actionable insights related to:

- Rider growth trends
- Revenue & profit analysis
- Seasonal performance
- Weather impact on riders
- Customer usage behavior
- Peak business hours
- Cost efficiency metrics

# Business Questions Solved in the Project

## Time-Based Usage Analysis:
1. Calculate the total riders per year, season, and hour of the day
2. Find the top 3 busiest days each year
3. Calculate Year-over-Year (YoY) rider growth
4. Compare average riders during holidays vs non-holidays
5. Compare weekday vs weekend rider behavior

## Rider Type & Weather Impact Analysis:
1. Find percentage contribution of each rider type
2. Analyze weather impact on riders
3. Calculate correlation between temperature and rider count
4. Compare riders on working vs non-working days

## Business & Revenue Analysis:
1. Calculate total revenue per year
2. Calculate yearly profit
3. Find yearly profit margin %
4. Determine the most profitable season
5. Identify years where rider growth exceeded revenue growth

## Advanced SQL Analysis:
1. Rank months by rider count within each year
2. Find top 5 busiest hours across all years
3. Identify highest rider day and weather condition per year
4. Calculate rolling 7-day average riders
5. Compare riders with previous day using LAG()

## Scenario-Based Business Analysis:
1. Estimate profit impact if price increases by 10%
2. Find days with riders above yearly average
3. Identify underperforming seasons
4. Calculate riders per dollar of COGS
